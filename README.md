# Lab Attendance System (研究室在室管理システム)

研究室メンバーの在室・不在状況をリアルタイムに記録・集計し、個人の活動実績を可視化するWebアプリケーションです。

## 🛠 技術スタック
- **Frontend**: Next.js (React), Tailwind CSS
- **Backend/DB**: Supabase (PostgreSQL)
- **Hosting**: Vercel

## 📝 主な機能

### 1. トップ画面（リアルタイムダッシュボード）
- **稼働状況の可視化**: 学年ごとにメンバーが自動ソートされ、各学年の「現在の在室人数 / 所属総人数」がひと目で分かります。
- **ワンタップ打刻**: 自分の名前の横にある「入室する」「退室する」ボタンから、直感的に状態を切り替えられます。
- **利用方法ガイド**: 初めて使うメンバーでも迷わないよう、ページ内にシンプルな操作説明を表示しています。

### 2. 本人確認・セキュリティ
- **サイト全体のアクセス制限**: 関係者以外の閲覧を防ぐため、サイト入場時に共通の暗証番号入力を要求します。
- **打刻時の本人認証**: 押し間違いやなりすましを防ぐため、打刻時に個人の4桁の暗証番号（PIN）の入力を求めます。
- **個人暗証番号の変更**: 各自の専用ページから、現在の暗証番号を用いた本人確認を経て、いつでも新しい暗証番号へ安全に変更できます。

### 3. 個人詳細ページ（活動実績の可視化）
- **活動データの自動集計**: 過去の打刻ログに基づき、「通算の来室日数」および「当日の在室時間」がリアルタイムに計算・表示されます。
- **打刻履歴の確認**: 自分が過去にいつ入退室したかのログを、個人ページ内でいつでも振り返ることができます。

### 4. 管理者専用画面（データのメンテナンス）
- **マスターパスワード認証**: 管理者用の共通暗証番号で保護された専用ダッシュボードを用意しています。
- **打刻ミスの救済機能**: メンバーの押し忘れや誤打刻があった場合、管理者が過去の入退室ログをその場で直接「修正」または「削除」できます。裏でデータが書き換わると、個人の集計日数や時間も自動で再計算されます。

## 📌 バージョン履歴
- **Ver 1.0**: 学年ごとにソートされたメンバーの一覧表示および基本的な入退室打刻システムを実装。
- **Ver 1.1**: サイト入場時に共通パスワードを要求するセキュリティ機能を実装。
- **Ver 2.0**: 打刻時の個人PIN認証、パスワード変更機能、個人ページでの来室日数・在室時間の自動集計、および誤打刻を修正・削除できる管理者画面、学年ごとのリアルタイムな在室数を実装。
